# Claude 进入 Microsoft Foundry 和 Microsoft 365 Copilot

> **原文**: [Anthropic News](https://www.anthropic.com/news/claude-in-microsoft-foundry)

## 核心内容

- Microsoft 和 Anthropic 宣布扩大合作，Claude 模型正式进入 Microsoft 生态系统
- Claude Sonnet 4.5、Haiku 4.5 和 Opus 4.1 在 Microsoft Foundry 公开预览
- Claude 为 Microsoft 365 Copilot 的 Researcher Agent 提供支持
- 企业客户可使用现有的 Azure 消费承诺 (MACC) 采购 Claude

## 解读

### Microsoft Foundry 整合

**可用模型**：
- Claude Sonnet 4.5
- Claude Haiku 4.5
- Claude Opus 4.1

**核心价值**：
- 企业可在现有 Microsoft 生态系统内构建基于 Claude 的生产应用和企业 Agent
- 支持与 Claude Code（Anthropic 的 AI 编程代理）配合使用
- 符合 MACC 资格，消除繁琐的供应商采购流程

### Microsoft 365 Copilot 整合

**Researcher Agent**：
- Claude 为复杂的多步骤研究任务提供支持
- 实现深度信息挖掘和分析

**Copilot Studio**：
- 支持使用 Claude 构建自定义 Agent
- Claude Opus 4.5 作为实验模型即将取代 Opus 4.1

**Excel Agent Mode**：
- 预览版支持使用 Claude 直接在 Excel 中构建和编辑电子表格
- 提升数据处理和分析效率

### 企业采购优势

**MACC 资格**：
- 已与 Microsoft 签订消费承诺的客户可直接采用 Anthropic 模型
- 无需单独开设供应商合同
- 对大型企业而言，可省去数月的采购周期

**数据处理保障（2026年1月起）**：
- Anthropic 成为 Microsoft 的子处理商
- 受 Microsoft 产品条款和数据处理附录 (DPA) 约束
- 不使用客户数据训练模型

### 与竞品对比

| 特性 | Claude in Copilot | OpenAI in Copilot |
|-----|------------------|-------------------|
| 模型选择 | Sonnet/Haiku/Opus | GPT 系列 |
| 编码能力 | 业界领先 | 强 |
| Agent 任务 | 优势场景 | 通用 |
| 办公任务 | 优势场景 | 强 |

Copilot 仍由 OpenAI 最新模型驱动，但客户现在可以灵活选择使用 Anthropic 模型，特别是在 Researcher 或 Copilot Studio 中构建 Agent 时。

### 对开发者/行业的影响

1. **模型选择权**：企业用户不再被单一模型锁定，可根据任务选择最优模型
2. **采购简化**：通过 Azure 统一采购，降低引入新模型的行政成本
3. **生态协同**：Claude 的编码和 Agent 能力与 Microsoft 办公生态深度结合
4. **竞争加剧**：打破了 OpenAI 在 Microsoft 生态的独占地位

### 可用性

- Microsoft Foundry：公开预览中
- Microsoft 365 Copilot Researcher：已上线
- Copilot Studio：支持自定义 Agent 开发
- Excel Agent Mode：预览中
- 地区限制：商业租户默认启用（欧盟和英国除外）
