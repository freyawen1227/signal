# Anthropic 发布 Claude Opus 4.5

> **原文**: [Anthropic - Introducing Claude Opus 4.5](https://www.anthropic.com/news/claude-opus-4-5)

## 核心内容

- **SWE-bench 首破 80%**：在真实 GitHub 代码修复任务上达到 80.9%，超越 GPT-5.1（76.3%）和 Gemini 3 Pro（76.2%），成为首个突破 80% 的模型
- **多语言编程领先**：SWE-bench Multilingual 8 种语言中 7 种排名第一
- **effort 参数**：新增 low/medium/high 三档计算力控制，medium 档位达到 Sonnet 4.5 同等性能，但 token 消耗减少 76%
- **大幅降价**：定价 $5/$25（输入/输出每百万 token），较上代 $15/$75 降幅达 66%
- **规格**：200K 上下文窗口，64K 输出限制，支持 Claude Chat、Claude Code、API 及三大云平台

## 技术解读

### SWE-bench 80.9% 意味着什么

SWE-bench Verified 不是合成测试，而是从 Django、Flask、Matplotlib 等真实 Python 仓库抽取的 GitHub Issues。80.9% 意味着：

- **10 个真实 bug 能修 8 个**：这是端到端的修复，包括理解 issue、定位代码、写出正确 patch
- **超越人类候选基准**：此前该榜单上人类工程师的通过率约在 75-78%
- **从「辅助」到「替代」的临界点**：过去 AI 编程是「帮你写代码」，现在是「帮你修 bug、做 code review、处理 PR」

### effort 参数的设计哲学

这是一个值得关注的产品设计：

| effort | 适用场景 | 成本 |
|--------|----------|------|
| low | 简单任务、快速响应 | 最低 |
| medium | 日常开发，性价比最优 | 中等 |
| high | 复杂推理、高难度任务 | 最高 |

**意义**：用户首次能显式控制「模型思考多久」，这是推理时计算（inference-time compute）产品化的信号。预计会成为行业标配。

### 定价策略分析

66% 降价不是让利，而是：

1. **规模效应**：Anthropic 推理基础设施成熟，成本下降
2. **市场策略**：拉开与 GPT-5 的性价比差距（GPT-5 目前 $10/$30）
3. **开发者锁定**：更低的价格 = 更多开发者构建 = 更高的切换成本

### 对开发者和创业者的影响

1. **AI 编程工具竞争加剧**：Cursor、Windsurf 等依赖模型能力，Opus 4.5 的领先意味着 Claude 生态的工具会更有优势
2. **Agent 可行性提升**：80% 的 bug 修复率意味着 AI Agent 在软件工程场景的可靠性达到可用门槛
3. **成本结构改变**：中小团队可以更激进地使用顶级模型，而不必在性能和成本间过度妥协
